# Website front-end brief for Codex

## Project
Build a premium marketing site for a digital product catalogue made of:
- Install Kit
- Agentic Engines
- Tuning Kits
- All Packs Bundle

The site should feel:
- engineered
- artistic
- contemporary
- slightly playful
- technically precise
- commercially clear

It should not feel like:
- a generic SaaS landing page
- a docs portal
- a dealership site
- a racing/tuner cliché
- a cold enterprise dashboard

## Visual direction
Use the attached mood board as the primary visual reference.
Ignore the brand name "Agent ECU".
Keep the visual language, not the literal naming.

Core visual cues to borrow:
- blueprint / graph-paper background logic
- isometric exploded system diagrams
- a hand-drawn or pen-sketch blueprint treatment so the site feels a little more playful and human
- thin technical linework with slight sketch irregularity rather than sterile CAD perfection
- restrained blue / white / navy palette
- minimal iconography
- terminal install-command treatment
- engineering annotations used sparingly
- permission / governance visuals as trust signals

## Design interpretation
This is not an automotive website in a literal sense.
It should borrow from:
- blueprint drawings
- workshop manuals
- engine architecture
- technical diagrams
- calibration / tuning language
- operating-system clarity
- hand-drawn product sketchbooks and annotated concept drawings

The metaphor is:
- Install Kit = foundation / ignition / deployment layer
- Agentic Engine = the actual operating engine
- Tuning Kit = optimisation / calibration layer
- Bundle = complete system

The important tonal balance is:
- engineered enough to feel credible
- sketched enough to feel playful
- precise enough to feel premium
- restrained enough to stay commercially clear

## Design goals
1. Make the products feel premium and ownable
2. Make the system feel easy to understand
3. Make the user trust the permission boundaries
4. Make the three-step ladder commercially obvious
5. Keep the page fun enough to feel designed, but restrained enough to feel serious

## Audience impression
The user should think:
- this feels intelligently designed
- this looks installable, not vague
- this has structure and boundaries
- this is more thoughtful than a prompt pack

## Suggested site structure
Single-page first pass is fine.
Sections should include:

1. Hero
2. Product selector / card row
3. Why it works
4. Permission / trust section
5. How it works
6. What’s inside the packs
7. CTA / bundle section

## Hero direction
Hero should feature:
- oversized headline
- short positioning subhead
- one strong central blueprint-style or exploded-architecture graphic
- subtle motion if possible
- a terminal-style install CTA block or secondary install cue

Suggested message territory:
- install once
- run structured AI workflows
- tune what comes next

The hero should feel like a system being assembled, not a generic app screenshot.

## Product selector section
Create a 4-card horizontal section across desktop.
This is a key commercial component.

Cards in order:
1. Install Kit
2. Workflow Pack
3. Skill / Tuning Pack
4. All Packs Bundle

### Important interaction
The second card should include a dropdown or segmented selector so the user can toggle between workflow variants.
For now include at least:
- Chief of Staff Agentic Engine
- Thought Leadership Agentic Engine

The card should update short copy when the workflow changes.

## Card content direction
Each card should sell benefits, not file formats.

### Install Kit card
Core benefit language:
- simple setup
- install Hermes or OpenClaw properly
- establish the right structure from the start
- reduce friction before the engine arrives

### Workflow Pack card
Core benefit language:
- install a workflow designed for a real use case
- adapt to the user rather than forcing a rigid template
- create a working operating system, not just prompts
- examples should switch based on dropdown selection

### Skill / Tuning Pack card
Core benefit language:
- improve the installed engine
- sharpen prompts, guardrails, and automation opportunities
- make the workflow more tailored and more capable
- treat this as calibration, not setup

### All Packs Bundle card
Core benefit language:
- the complete system
- foundation, workflow, and tuning together
- best value / most complete path
- one purchase, full stack

## Suggested card behaviour
- each card should feel like a product object, not a generic pricing tile
- subtle hover movement is good
- line art or icon treatment should distinguish the cards
- use one primary CTA per card
- bundle card can be slightly more visually dominant

## Trust / permission section
Use the permission-matrix idea from the mood board, but translate it into a more commercial website component.
Do not use the sample mood-board matrix content literally.

Purpose of this section:
- show that the system does not go rogue
- show that approval boundaries are designed in
- turn permission control into a selling point

Possible format:
- clean matrix
- layered capability graphic
- controlled-state cards
- diagram showing read / draft / execute / approval states

## Why it works section
Explain the three-part logic:
- first install the runtime properly
- then install the workflow
- then tune the workflow

This should feel like engineered sequencing.
Not like upsell stacking.

## Styling rules
Typography:
- contemporary sans serif for headings and body
- monospace only for terminal/code details
- avoid luxury serif clichés

Palette:
- off-white / white background
- pale blueprint blue
- dark navy / ink blue
- very limited accent usage

Linework and illustration:
- diagrams should feel like premium pen-and-blueprint sketches, not sterile engineering software exports
- preserve technical precision, but allow slight hand-drawn irregularity in strokes, arrows, and annotation treatment
- sketch quality should add charm and distinctiveness, not mess

Layout:
- generous whitespace
- strong grid discipline
- occasional full-bleed diagram sections
- avoid clutter

Motion:
- subtle, purposeful, mechanical-feeling motion
- layer reveals
- line-draw or diagram animation
- no flashy startup animation tropes

## Avoid
- generic AI gradients
- neon cyberpunk
- carbon fibre or racing clichés
- literal sports-car imagery
- dashboard overload
- over-annotation everywhere
- making the site feel like technical documentation only

## Tone of copy
Copy should feel:
- clear
- premium
- practical
- designed
- calm

Avoid:
- hype
- vague AI promises
- startup clichés
- overclaiming automation
