# Codex brief — Agentic Engines website, simplified layout pass

## Project
Revise the existing Agentic Engines homepage again.

This pass is about simplifying the layout and information hierarchy.
Use https://www.octoai.space/ as the exact reference for layout simplicity, page rhythm, and pricing presentation.
Do not use a different "Octo" or similarly named product as the reference.

Do not copy Octo visually.
Do not copy Octo’s brand voice.
Do not copy Octo’s product category.

Use it only as a reference for:
- a simpler landing-page structure
- a calmer, more direct information flow
- faster comprehension
- cleaner pricing tier presentation
- less section sprawl

The Agentic Engines site must still preserve its own identity:
- blueprint blue on chalk white
- sketched blue-pen / paper-stack aesthetic
- playful but controlled
- modular, snap-together system feel

## Core objective
Make the site feel much simpler, more playful, and more immediately understandable.

The current prototype is still too complicated layout-wise.
It still reads too much like a proposition document.

This next pass should feel closer to:
- a clean, modern product landing page
- fewer sections
- more useful information above the fold
- faster scanning
- stronger workflow choice and bundle logic early
- less explanatory repetition

## The one thing the page must make clear immediately
**Stop chatting with AI. Install a working agent in one command. No engineering needed.**

That remains the locked hero line.

## Layout simplification reference
Study Octo AI for these layout traits:
- immediate proposition above the fold
- short supporting copy
- product proof visible early
- pricing presented cleanly and directly
- simple vertical rhythm
- fewer conceptual detours
- more “yes, I get it” within seconds

Do not reproduce Octo’s black visual style.
Translate its simplicity into the Agentic Engines blueprint / paper world.

## Visual direction
Keep the design world, but simplify it.

Preserve:
- blue pen on white paper
- blueprint precision
- paper-stack layering
- sketched diagrams
- cut-out paper states
- tactile buttons and cards

Simplify:
- reduce diagram complexity
- reduce secondary annotation noise
- reduce visual clutter
- reduce the number of competing objects on screen

Add more playfulness through:
- paper cut-out layering
- simple snapped-together component compositions
- selected states that feel like movable paper pieces
- a clearer, lighter page rhythm

The site should feel like:
- a sketched system laid out on a desk
- pieces of a workflow snapping together
- a product page that is fun to browse

It should not feel like:
- an enterprise dashboard
- a strategic framework presentation
- a dense product spec sheet
- a shiny SaaS template

## Content priority
Push as much useful information as possible above the fold.

Within the hero and first scroll, the user should understand:
- what this is
- how it is different from generic chat AI
- that they can choose a workflow pack for a specific job
- that the full bundle is the simplest buy
- that setup is easy

## Structural direction
Simplify the page structure.
Aim for a layout closer to this:

1. Hero
2. Workflow Pack chooser
3. Bundle / pricing section
4. How it works
5. Trust / boundaries
6. Final CTA

If any existing sections overlap with these, merge or remove them.
Do not preserve sections just because they already exist.

## Hero
Keep the locked headline.

The hero should contain:
- headline
- one short subhead
- a clean pricing block
- primary CTA
- secondary CTA
- a short proof line
- one stronger hero object that acts like a product-photo-equivalent moment inside the sketch language

Hero subhead direction:
Short, direct, and compressed.
Closer to:
- Pick a workflow. Install the base system. Tune it to your job.
- Choose the job you want AI to do, then install the full system around it.

Do not over-write the subhead.

Above-the-fold pricing should make the bundle value obvious fast.
Use the $25 vs $34 contrast clearly.

## Workflow Pack chooser
This should be immediately below the hero.
It is one of the main selling sections.

It should feel simple, useful, and concrete.
Frame it around:
**Choose the job you want AI to do.**

Workflow options for now:
- Chief of Staff
- Thought Leadership
- Solo Operator

Each option should answer quickly:
- what changes for me
- what kind of work it helps with
- what kind of rhythm or output it creates

Use compact, outcome-led language.

Examples of tone:
- Chief of Staff: Run your week with less chasing and better follow-through.
- Thought Leadership: Turn rough ideas into structured posts, themes, and publishing momentum.
- Solo Operator: Build a lighter rhythm for planning, prioritising, and shipping without losing context.

Do not over-explain.
Do not make this section feel abstract.

Important implementation requirement:
The Workflow Pack selector must remain easy to extend later as more workflow packs are added.
This is a system, not a two-option one-off.

## Bundle / pricing section
Condense the current product ladder and bundle rationale into one clean section.

This section should explain simply:
- Install Kit = sets up the base system
- Workflow Pack = gives it a real job
- Tuning Kit = shapes it to how you work
- Bundle = $25
- Separate total = $34

The section should feel obvious, not argumentative.
It should not sound like a strategist explaining why the bundle makes sense.
It should simply make the bundle feel like the cleanest place to start.

Use a simple tier/card presentation inspired by Octo’s pricing clarity.

Include:
- a visually dominant bundle card or pricing block
- clear separate prices
- immediate contrast between separate total and bundle total
- strong primary CTA on the bundle

## How it works
Keep this short.
Three steps only.
Fast to scan.

Suggested structure:
1. Run one command
2. Connect your model
3. Paste the prompt

Keep the setup command as a visual proof object, but do not let it dominate the page.
It should support simplicity, not become the main attraction.

## Trust / boundaries
Keep the idea, but simplify the presentation.

The user should quickly understand:
- helpful does not mean uncontrolled
- approvals stay visible
- the system is designed to feel safe and legible

This section should scan quickly.
Avoid heavy prose.
Avoid internal-sounding terminology.

## Final CTA
Close simply.
Repeat the bundle CTA.
Offer a secondary path to browse Workflow Packs.

## Tone
The tone should be:
- calm
- simple
- direct
- plain English
- product-confident
- playful in a controlled way
- light, not clever-clever

Avoid:
- internal strategy language
- technical jargon
- defensive explanation
- pseudo-philosophical copy
- startup hype

## Language rules
Use more:
- install
- build
- snap together
- baseplate
- workflow
- pick the job
- make it yours
- one command
- one prompt
- smooth
- simple
- working agent

Avoid:
- belief engine
- commercially obvious
- compliance theatre
- orchestration layer
- infrastructure
- autonomous system
- full automation
- magic
- revolutionary

## Platform requirement
Keep the implementation:
- static HTML/CSS/JS
- modular
- clean enough to adapt into Shopify later
- free of unnecessary framework complexity

## Success standard
The page should feel like something a first-time visitor can understand in under 20 seconds.
It should feel simpler, lighter, more playful, and more buyable.
It should still feel like Agentic Engines, not Octo.

Do the work directly.
Compress, merge, remove, and simplify.
