# Codex brief — Agentic Engines website, pass two

## Project
Refine the existing Agentic Engines front-end prototype into a sharper second pass.

This site is for a self-serve, low-cost AI product system that helps curious, relatively novice AI users move beyond blank-box AI and into simple, working agent systems built for real jobs.

This is not an enterprise AI dashboard.
This is not a generic SaaS landing page.
This is not a technical docs portal.

It should feel like a simple, satisfying, modular way into agentic working.

## Core commercial objective
The commercial goal is to sell the **$25 Starter Bundle**.

Pricing ladder:
- Install Kit / Baseplate = $9
- Workflow Pack = $15
- Tuning Kit = $10
- Bought separately = $34
- Starter Bundle = $25

Use this pricing gap deliberately.
The bundle should feel like the obvious, low-friction choice.

## Key persuasion sequence
The homepage must make these beliefs land in order:
1. This is easier than I expected.
2. This is more useful than just chatting with ChatGPT.
3. One of these Workflow Packs fits what I want to do.
4. The bundle is the obvious choice.

Important:
- the bundle is the commercial goal
- but the key belief moment is the **Workflow Pack** section
- if the user believes setup is simple and believes one Workflow Pack clearly fits their goals, the Install Kit and Tuning Kit should feel like the obvious surrounding pieces that make the bundle an easy decision

## Locked hero line
Use this exact hero headline:

**Stop chatting with AI. Install a working agent in one command. No engineering needed.**

## Brand / product metaphor
The site should frame the product family as **snap-together AI**.

Think Lego, but interpreted in a sleek, grown-up, blueprint-sketch way.
Do not make it childish or toy-like.

Product metaphor:
- Install Kit = the baseplate
- Workflow Pack = the build
- Tuning Kit = the custom pieces
- Bundle = the full build

Emotional tone:
- a weekend project that actually works on Monday
- satisfying to assemble
- modular
- approachable
- structured

## Audience
Target audience:
- relatively novice AI users
- already AI-curious
- may use ChatGPT Plus, Gemini, or similar tools
- fascinated by AI, but bored of typing into a blank box
- find tools like Codex or Claude Code intimidating
- want something real, but do not want a technical weekend swallowed by setup friction

## Core problem to dramatise
Do not lead with enterprise risk.
Lead with the boredom and limitation of current AI use.

Current experience to dramatise:
- blank-box prompting
- copy-paste fatigue
- repetitive prompting
- generic output
- forgotten context
- no sense that the AI actually has a job

The site should frame Agentic Engines as the antidote to that.

Contrast:
- Old AI = blank-box prompting, copy-paste fatigue, generic output, no real workflow
- Your AI = installed once, built for a job, shaped around your rules, remembers the structure, designed to work with you

## Core promise
This site should not promise magic.
It should not promise miracles.
It should not imply risky, uncontrolled autonomy.

It should promise:
- a cleaner entry into AI agents
- simple setup
- a workflow that already knows its job
- better structure
- better memory and role clarity
- a better relationship between user and agent

## Visual direction — simplify the first pass
The first pass was too concept-heavy and slightly too gadget-like.
Simplify it.

Primary visual direction:
- blue pen on white paper
- sketch-like blueprint precision
- hand-drawn workflow diagrams
- modular page composition
- cut-out white paper states layered over a larger page
- light, intelligent, satisfying, editorial, and approachable

The page should feel like:
- a sketched workflow system being assembled in real time
- a designed notebook / blueprint / technical sketch page
- modular parts snapping into place

It should not feel like:
- an over-rendered product advert
- a futuristic gadget site
- a car dashboard UI clone
- a gaming interface
- a docs site in disguise
- a hard industrial control panel

Responsive rule:
- desktop is the priority, but the site must still work cleanly on mobile
- ensure the blueprint aesthetic, terminal cards, and Workflow Pack selector stack cleanly and remain legible on a small mobile screen
- do not let sketch elements, annotations, or layered paper states break the mobile viewport

## Color direction
Prioritise:
- white / chalk white background
- blueprint blue as primary line and accent color
- restrained dark ink / navy for text
- use any brighter accent very sparingly, only where it genuinely helps state or CTA emphasis

The overall feel should still be mostly blue pen on white paper.

## Typography direction
Use:
- a clean geometric sans-serif for display and body
- a precise monospace for terminal/code elements only

Do not choose something that feels generic, lifeless, or overly corporate if a slightly more ownable alternative is available.
But do not overcomplicate typography.

## UI treatment direction
Use these ideas:
- sketched blueprint diagrams
- hand-drawn arrows / annotation logic with restraint
- cut-out paper button states
- selected states that feel like white paper pieces placed over the page
- modular cards that feel like product objects
- terminal cards as proof moments
- permission / trust section as controlled reassurance, not compliance theatre

Interaction rule:
- include explicit CSS `:hover` and `:active` states
- buttons and selected cards should feel tactile, like a piece of paper being pressed down or slightly lifted
- use subtle drop shadows, crisp borders, and restrained motion
- avoid soft digital blur as the main interaction language

Important:
- keep as much genuinely useful information above the fold as possible
- the page should not be atmosphere-first
- the hero should deliver comprehension, pricing logic, and setup simplicity very quickly

## Homepage structure
Build or refine the page in this order:

1. Hero
2. Why this exists
3. Product ladder
4. Workflow Pack selector
5. Trust / permission reassurance
6. How setup works
7. Bundle rationale
8. Final CTA

## Section-by-section requirements

### 1. Hero
Must include:
- locked hero headline
- a clear supporting subhead
- a strong pricing cue above the fold
- primary CTA for the bundle
- secondary CTA to browse kits
- a visual that supports the modular, sketched, blueprint-based product system
- a simplicity proof line such as: one command, one model hookup, one prompt

Suggested subhead direction:
- You don’t need to be technical to move beyond the blank box. You just need the right baseplate, a working workflow, and a quick tune-up.

Primary CTA:
- Get the Full System — $25

Secondary CTA:
- Browse the Kits

The above-the-fold area must make the offer understandable fast.

### 2. Why this exists
Headline direction:
- AI is impressive. Most people still use it the boring way.

This section should explain that most people are still:
- typing into the same blank box
- starting from scratch every time
- getting repetitive, generic results
- never turning AI into a real workflow

Agentic Engines changes that by installing a system with:
- structure
- memory
- a role
- clearer boundaries

### 3. Product ladder
Use the snap-together logic.

Four cards:
1. The Baseplate — Install Kit — $9
2. The Build — Workflow Pack — $15
3. The Custom Pieces — Tuning Kit — $10
4. The Full Build — Starter Bundle — $25

Each card must sell benefits, not file formats.

#### Install Kit card
Sell:
- simple setup
- one command to get the environment right
- removing setup friction before it becomes a problem

#### Workflow Pack card
Sell:
- a working agent built for a real job
- clear relevance to user goals
- role-shaped usefulness rather than blank-box prompting

#### Tuning Kit card
Sell:
- making it yours
- replacing generic AI habits with the user’s own voice, rules, and working style
- refining prompts, guardrails, habits, and useful automation

Avoid phrases like:
- eradicate default AI behavior

Prefer phrases like:
- make it yours
- replace generic AI habits with your rules
- tune the system until it works more like you do

#### Bundle card
Make this visually dominant.
It should feel like the obvious deal.

Visually anchor the discount.
Show the combined separate price with a strikethrough next to the bundle price so the value is obvious immediately.
Example pattern:
- ~~$34~~ $25

Sell:
- everything snapped together
- the easiest route from setup to working agent to tailored workflow
- lower total cost than buying separately
- strongest value / clearest path

### 4. Workflow Pack selector
This is the most important belief section on the site.

The user must come away thinking:
**Yes, this one is for me.**

For now include at least:
- Chief of Staff Agentic Engine
- Thought Leadership Agentic Engine

The card / selector must update:
- title
- short copy
- benefit bullets
- CTA label if useful

Chief of Staff framing:
- turn scattered notes, meetings, tasks, and follow-ups into a cleaner operating rhythm
- built for people who want help keeping the right things moving

Thought Leadership framing:
- turn rough notes, voice memos, and half-formed ideas into a repeatable content workflow
- built for people who want AI to help shape better thinking, not generic posts

Scalability requirement:
- build the component architecture so additional Workflow Packs can be added quickly later
- do not design this as a fixed two-option component
- the selector should be easy to extend without redesigning the section

### 5. Trust / permission reassurance
Headline direction:
- Built to be useful. Designed not to drift.

This section should reassure the user that:
- a working agent should feel like a better relationship, not a loss of control
- the system has clear boundaries around read, draft, and approval states
- structure is part of the product, not an afterthought

Do not sound corporate, paranoid, or compliance-heavy.
This is reassurance, not enterprise security copy.

### 6. How setup works
This section is critical.
It must make simplicity feel believable.

Headline:
- From blank box to working agent in three steps.

Steps:
1. Run one command
2. Connect your model
3. Paste the prompt

Visual requirement for step 1:
- include a minimalist terminal-style card containing a realistic setup command
- this should act as immediate visual proof of simplicity
- example pattern: `curl -fsSL https://agenticengines.com/install.sh | bash`

Supporting line:
- If you know how to copy and paste, you can do this.

This section should be visual, fast to read, and confidence-building.

### 7. Bundle rationale
Headline direction:
- The easiest way in is the full build.

This section should make the pricing logic and sequencing obvious:
- Install Kit gets the environment right
- Workflow Pack gives the agent a real job
- Tuning Kit makes it fit you better

The user should feel:
Why would I buy these separately when the bundle is so clearly the better deal?

### 8. Final CTA
Headline direction:
- Your weekend project is ready. Start simple. Build something real.

Primary CTA:
- Download the Starter Bundle — $25

Secondary CTA:
- Browse Workflow Packs
