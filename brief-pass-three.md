# Codex brief — Agentic Engines website, pass three

You are revising the live prototype homepage for Agentic Engines.

Your job is not to add more thinking. Your job is to make the page simpler, faster to understand, and more commercially effective.

We are now in pass three.

## Context
The current prototype is directionally right. It has a stronger blue-pen-on-white-paper feel, the hero is improved, the pricing logic is clearer, and the Workflow Pack concept is now much more viable. However, the page still feels too much like a smart proposition document turned into a website. It explains too much, repeats adjacent arguments across too many sections, and takes too long to surface the main sell.

The next pass should be a creative edit, not another broad rewrite.

## Primary objective
Make the proposition, workflow choice, and buying logic obvious within the first screen and first scroll.

## What the page must now do better
- get more of the message higher up
- simplify the structure
- reduce repeated explanation
- feel more like a product page and less like a strategic rationale
- keep the strongest ideas, but express them with fewer words
- improve emotional pull without losing clarity
- preserve the sketch / paper / blue-pen world, but make the top of the page feel more iconic and more immediately desirable

## What is already working and should be preserved
- the core headline territory: “Stop chatting with AI. Install a working agent in one command. No engineering needed.”
- the Starter Bundle logic: $25 bundle vs $34 separately
- the product architecture: Install Kit + Workflow Pack + Tuning Kit
- the Workflow Pack concept as the main differentiator
- the idea that this is not blank-box AI, but a pre-shaped working system
- the simple install / setup logic
- the trust logic around boundaries, approvals, and visible control

## What is not working and must be fixed
- too many sections making similar arguments
- too much explanatory copy
- some language sounds like internal strategy language rather than customer-facing product copy
- the strongest selling mechanic, the Workflow Pack choice, appears too low
- the bundle logic is good but still spread across too many sections
- the hero is clearer than before but still lacks one stronger “product-photo-equivalent” visual moment
- the page still invites agreement more than desire

## Tone direction
The tone should be:
- calm
- simple
- direct
- product-confident
- practical
- plain English
- light, not clever-clever
- not startup-hype
- not pseudo-philosophical
- not self-conscious

Avoid language that feels like internal strategic commentary, such as:
- belief engine
- commercially obvious
- compliance theatre
- sensible default, not just a pricing trick
- any similar phrasing that sounds like a strategist explaining the sales logic instead of a product confidently presenting itself

## Core instruction
Do not add more sections.
Do not add more theory.
Do not expand the copy.
Compress, merge, reorder, and sharpen.

## Structural direction
Rebuild the homepage to follow this order:

1. Hero
   Keep the core headline.
   Shorten the subcopy.
   Make the pricing and bundle logic visible immediately.
   Keep the install command or equivalent proof element, but it should support the proposition rather than dominate it.
   Ensure the top of the page communicates:
   - what this is
   - how it is different
   - the easiest way to buy

2. Workflow Pack section
   Move this much higher, ideally immediately below the hero.
   This should become one of the main selling sections on the page.
   The framing should make clear that the customer is choosing the job they want AI to do for them.
   The section should feel concrete and useful, not abstract.

3. Bundle section
   Condense the current product and pricing logic into one clean commercial section.
   It should clearly explain:
   - Install Kit = sets up the base system
   - Workflow Pack = gives it a real job
   - Tuning Kit = shapes it to how you work
   - $25 bundle
   - $34 separately
   This section should sell the bundle as the cleanest place to start, but without over-explaining or sounding defensive.

4. How it works
   Keep this short.
   Three steps only.
   Fast to scan.
   This is there to reassure, not to dominate.

5. Trust / boundaries
   Keep the core idea, but simplify heavily.
   The user should quickly understand:
   - helpful does not mean uncontrolled
   - actions and approvals stay legible
   - the system is designed to feel safe and visible

6. Final CTA
   Repeat the bundle CTA and provide a secondary browse-packs path.

## Content priorities
The page must communicate these ideas earlier and more clearly:
- this is not generic chat AI
- you are buying a shaped workflow, not just a setup kit
- the Workflow Pack is what makes the system feel useful from day one
- the bundle is the fastest and simplest way in
- control and boundaries are part of the product design

## Copy direction for the hero
Do not lose the strength of the current headline.
But make the supporting copy tighter and more immediate.

Aim for a structure like this:
- headline
- one short subhead
- pricing block
- primary CTA
- secondary CTA
- one short proof strip or install reassurance line

The subcopy should feel closer to:
“Pick a workflow. Install the base system. Tune it to your job. No engineering needed.”
You do not have to use that exact line, but stay in that territory.
Keep it compressed.

## Copy direction for the Workflow Pack section
This section should feel like the heart of the product.
Frame it around choosing the job you want AI to do.
Make each pack description more concrete and outcome-led.
Each pack should quickly answer:
- what changes for me
- what type of work this helps with
- what kind of output or rhythm it creates

For example, the style of description should be closer to:
Chief of Staff:
Run your week with less chasing and better follow-through.
Turns notes into briefs, actions, and decision-ready summaries.

Thought Leadership:
Turn rough ideas into structured posts, themes, and publishing momentum.

Solo Operator:
Build a lighter operating rhythm for planning, prioritising, and shipping without losing context.

Do not over-write these. Keep them compact.

## Copy direction for the bundle section
Condense all current bundle / ladder / rationale material into one section.
The section should not feel argumentative.
It should feel simple and obvious.
It should explain the parts and the price difference cleanly, then move on.

## Copy direction for the trust section
Simplify the language substantially.
This section should be lighter, cleaner, and easier to scan.
A visual or mini-state model is fine.
Heavy explanatory prose is not.

## Visual direction
Preserve the existing blue-pen-on-white-paper, sketch-board, paper-stack language.
Do not drift back toward shiny SaaS dashboard visuals.
But create one stronger hero object or visual proof moment that feels like the equivalent of a product photograph inside this sketch language.
This could be:
- a cleaner assembled system stack
- a stronger workflow card object
- a more iconic before/after moment
- a sharper snap-together component composition
It should make the product feel more tangible and desirable immediately.

## Editing rules
- cut repetition aggressively
- merge sections where arguments overlap
- shorten paragraphs
- favour scan-ability
- remove anything that sounds like rationale for the team rather than value for the buyer
- keep the page commercially legible
- make the first screen do much more work

## Output required
1. A revised homepage structure
2. Revised section-by-section copy
3. Any section removals or merges clearly implemented in the rewrite
4. If useful, minor markup / component adjustments to support the new hierarchy
5. Keep the prototype adaptable to later Shopify implementation
6. Preserve the static HTML/CSS/JS approach

## Success standard
The revised page should feel like something a first-time visitor can understand in under 20 seconds.
It should feel simpler, more confident, and more purchasable.
It should create more immediate desire while retaining the current clarity and logic.

Do the work directly.
Do not just critique.
Do not return recommendations only.
Rewrite the page.
